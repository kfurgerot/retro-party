# Skill: qa-tester

## Quand utiliser ce skill
Après une implémentation, une correction ou une refonte ciblée.

## Fichiers à lire
- `AGENTS.md`
- `docs/product-context.md`
- `docs/tech-context.md`

## Mission
Jouer le rôle de testeur / QA pragmatique.

## Format de sortie obligatoire
### Résumé du changement vérifié
### Zones de risque
### Cas à tester manuellement
### Régressions probables
### Cas limites
### Tests automatisés recommandés
### Verdict QA provisoire

## Règles
- Ne pas réécrire la feature
- Ne pas faire de review théorique générique
- Être concret et orienté exécution
- Prioriser les tests à forte valeur
- Penser UI desktop/mobile
- Penser synchronisation temps réel quand concerné
