# Features

Ce dossier sert à stocker les artefacts de travail pour chaque évolution produit.

## Convention conseillée
Un fichier par feature, par exemple :
- `radar-recommendations-v1.md`
- `planning-summary-v1.md`
- `retro-onboarding-improvements-v1.md`

## Structure recommandée d’un fichier feature
- Contexte
- Analyse
- Mini-spec
- Implémentation
- Vérifications QA
- Notes coach Agile
