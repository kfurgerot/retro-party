## Contexte produit

AgileSuite est une application SaaS destinée aux Scrum Masters, Agile Coaches, Product Owners, managers et équipes de développement.

L’objectif est de proposer une suite de modules agiles simples, visuels, collaboratifs et engageants pour animer, diagnostiquer, mesurer et améliorer les équipes.

L’application contient actuellement plusieurs modules :

1. Retro Party

- Module d’amélioration continue.
- Rétrospective gamifiée sous forme de plateau de jeu.
- Les participants avancent, répondent à des questions, génèrent des idées d’amélioration et collectent des points / kudos.
- Objectif : rendre les rétrospectives plus engageantes, moins répétitives et plus utiles.

2. Planning Party

- Module d’estimation collaborative.
- Inspiré du Planning Poker.
- Permet de préparer des éléments à estimer, lancer une session, voter, afficher une synthèse des votes, identifier les écarts et faciliter la discussion.

3. Radar Party

- Module de diagnostic d’équipe.
- Permet de créer des questionnaires par thèmes, faire répondre les membres d’une équipe, puis afficher un radar individuel et collectif.
- Objectif : visualiser la maturité, les zones fortes, les zones faibles et les écarts de perception dans l’équipe.

4. Matrice de compétences

- Module pour mesurer les compétences d'une équipe.
- Permet de cartographier les compétences d’une équipe.
- Chaque membre peut déclarer son niveau, son envie de progresser et sa capacité à aider/former les autres.
- Objectif : identifier les forces, faiblesses, dépendances et opportunités de montée en compétence.

## Parcours clés à concevoir

1. Accueil / landing interne

- Présenter clairement AgileSuite.
- Montrer les modules disponibles.
- Donner envie de lancer rapidement une activité.
- Mettre en avant les bénéfices : mesurer, améliorer, estimer, développer les compétences.

2. Dashboard utilisateur connecté

- Voir ses équipes.
- Voir ses sessions récentes.
- Voir ses templates préparés.
- Accéder rapidement aux modules.
- Créer une nouvelle session.

3. Choix d’un module

- Chaque module doit avoir une identité claire.
- Les modules doivent être regroupés par catégories :
  - Mesurer
  - Améliorer
  - Estimer
  - Développer l’équipe
- Chaque module doit proposer des actions simples :
  - Lancer une session rapide
  - Préparer une session
  - Rejoindre une session
  - Voir les résultats / historiques

4. Préparation d’une session

- Retro Party : préparer les questions, thèmes, règles, durée.
- Planning Party : préparer les items à estimer, système de vote, participants.
- Radar Party : préparer les thèmes, questions, échelle d’évaluation.
- Matrice de compétences : créer les catégories, compétences, équipe, niveaux.

5. Session live

- Interface claire pour l’animateur.
- Interface simple pour les participants.
- Code de session facile à partager.
- État temps réel lisible.
- Expérience fluide desktop et mobile.

6. Résultats

- Synthèse claire.
- Insights actionnables.
- Exports possibles.
- Mise en avant des décisions, actions et apprentissages.
